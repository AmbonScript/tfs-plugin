The /%1$s/ endpoint

Security

When your Jenkins is secured, you can use HTTP BASIC authentication to authenticate remote API requests. See Authenticating scripted clients for more details.

WARNING

This endpoint SHOULD be reachable only through a secure channel.

Build commands

The following build commands are available to PUT to: %2$s
Name Path Sample payload

Example

Suppose we have a secured Jenkins server at %3$s and a dedicated Remote Trigger user named remote that has an API Token of 92e9d8998a8c005697d252f09a2a311b. The following example demonstrates a PUT to the ping command for the job named JOB_NAME using the curl tool: 
curl --request PUT %3$s%1$s/ping/JOB_NAME --data-urlencode json='{"parameter": [{"name":"id", "value":"123"}, {"name":"verbosity", "value":"high"}], "team-parameters": {"name":"value"}}' --user remote:92e9d8998a8c005697d252f09a2a311b
The console output should look like the following when the ping command from the %1$s endpoint returns the provided payload, formatted for compactness: 
{"parameter":[{"name":"id","value":"123"},{"name":"verbosity","value":"high"}],"team-parameters":{"name":"value"}}
